Firebase Hostingのセキュリティ

エンタープライズレベルのセキュリティで、あなたのウェブサイトとユーザーを保護します。Google Cloudの最先端セキュリティ技術を活用し、包括的な保護を提供します。

セキュリティ機能

Firebase Hostingが提供する包括的なセキュリティ機能をご紹介します

自動SSL証明書

Let's Encryptを使用した無料のSSL証明書を自動で発行・更新。ドメイン検証済み証明書により、全ての通信を暗号化します。

  • 自動発行・更新
  • カスタムドメイン対応
  • ワイルドカード証明書

HTTPS強制

すべてのHTTPリクエストを自動的にHTTPSにリダイレクト。Transport Layer Security (TLS) 1.3を使用した最新の暗号化技術。

  • TLS 1.3対応
  • HSTS設定
  • 自動リダイレクト

DDoS保護

Google Cloudの分散拒否サービス攻撃（DDoS）保護により、大規模な攻撃からサイトを守ります。

  • 自動検出・防御
  • リアルタイム監視
  • 無制限の帯域幅保護

Firebase Authentication連携

Firebase Authenticationと統合し、多要素認証、OAuth、匿名認証などの高度な認証機能を提供。

  • 多要素認証（MFA）
  • ソーシャル認証
  • カスタム認証

セキュアCDN

世界中に配置されたセキュアなエッジロケーションから、暗号化されたコンテンツを高速配信。

  • 120+ エッジロケーション
  • キャッシュ暗号化
  • オリジン保護

セキュリティ監視

24時間365日のセキュリティ監視とリアルタイムアラート。不正アクセスや異常な活動を即座に検出。

  • リアルタイム監視
  • 異常検知
  • セキュリティレポート

セキュリティベストプラクティス

Firebase Hostingでセキュリティを最大化するための推奨事項

1

セキュリティルールの設定

Firebase Security Rulesを適切に設定し、未認証ユーザーからのアクセスを制限します。最小権限の原則に従い、必要最小限のアクセス権限のみを付与してください。

2

定期的なセキュリティ監査

Firebase Consoleのセキュリティダッシュボードを定期的に確認し、異常なアクティビティや潜在的な脅威を監視します。セキュリティアラートを有効にして即座に通知を受け取りましょう。

3

強力な認証の実装

Firebase Authenticationの多要素認証（MFA）を有効にし、パスワードポリシーを強化します。OAuth 2.0とOpenID Connectを使用したソーシャルログインも併用しましょう。

4

セキュリティヘッダーの設定

Content Security Policy (CSP)、X-Frame-Options、X-Content-Type-Optionsなどのセキュリティヘッダーを適切に設定し、XSS攻撃やクリックジャッキングを防止します。

5

データ暗号化

機密データは転送時と保存時の両方で暗号化します。Firebase Hostingは自動的にHTTPSを使用しますが、カスタムAPIエンドポイントでも同様の保護を実装してください。
