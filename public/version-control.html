バージョン管理

デプロイ履歴の管理とワンクリックロールバック機能で安全な運用を実現

バージョン管理概要

Firebase Hostingのバージョン管理システムは、すべてのデプロイを自動記録し、 問題が発生した際に即座に前のバージョンにロールバックできます。 Git のような感覚でWebサイトのバージョンを管理できます。

🕐 タイムマシン機能

過去のどのバージョンにも瞬時に戻ることができます。 新機能のリリースで問題が発生しても、1クリックで安定したバージョンに復元可能です。

バージョン管理機能

デプロイ履歴管理

すべてのデプロイが時系列で記録され、 詳細な変更内容とメタデータを確認できます。

  • 完全なデプロイ履歴
  • 変更ファイル一覧
  • デプロイ時刻記録
  • ユーザー・コメント記録

ワンクリックロールバック

過去の任意のバージョンに瞬時にロールバック。 複雑な手順は不要で、緊急時も安心です。

  • 即座のバージョン切り替え
  • 部分ロールバック対応
  • ロールバック履歴記録
  • 確認プロセス付き

変更差分表示

バージョン間の詳細な差分を視覚的に確認。 何が変更されたかを瞬時に把握できます。

  • ファイル差分表示
  • 追加・削除・変更の識別
  • サイズ変更追跡
  • バイナリファイル対応

バージョンタグ

重要なリリースにタグを付けて管理。 「v1.0.0」「production」などの分かりやすい名前で整理できます。

  • カスタムタグ設定
  • セマンティックバージョニング
  • リリースノート添付
  • タグ検索機能

プレビュー機能

ロールバック前に過去のバージョンをプレビュー。 安全性を確認してから本番反映できます。

  • バージョンプレビューURL
  • A/Bテスト対応
  • ステージング環境
  • テスト環境自動生成

変更通知

デプロイやロールバック時の自動通知。 チーム全体で変更を把握できます。

  • Slack・Discord連携
  • メール通知
  • Webhookサポート
  • カスタム通知ルール

バージョン管理ワークフロー

1

自動バージョン作成

firebase deployでの新しいバージョンが自動作成されます

firebase deploy --message "v1.2.0 - Add new feature"
2

バージョン履歴確認

Firebase Consoleでデプロイ履歴と変更内容を確認

Hosting > リリース履歴 > バージョン詳細
3

問題検知時のロールバック

問題が発生した際は即座に安定バージョンにロールバック

Console > "ロールバック" ボタン > 確認
4

修正後の再デプロイ

問題修正後、新しいバージョンとして再デプロイ

firebase deploy --message "hotfix: Fix critical bug"

高度なバージョン管理

CLI コマンド

履歴表示 firebase hosting:releases:list
バージョン詳細 firebase hosting:releases:view [ID]
ロールバック firebase hosting:releases:rollback [ID]
クローン firebase hosting:releases:clone [ID]
